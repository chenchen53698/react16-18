redux在设计上的一些缺陷:
1.基于getstate获取的公共状态，是直接和redux中的公共状态，共同相同的堆地址，这样导致,可以直接修改公共状态信息.
  需要每次都克隆一下避免直接修改: state = _.clone(true, state);
2.让组件更新的办法放在事件池中，当公共状态改变,会通知时间池中的所有方法执行！！
此操作：放置方法的时候，没有办法设置状态的依赖，这样，后期不论哪个状态被修改，事件池中的所有方法都要执行(相关的组件都要进行更新)！！！
真实项目中：由于react-router让很多组件都释放了，只展示当前模块的spa，这样即便组件更新的方法执行，因为组件释放了也不会产生太大的影响
